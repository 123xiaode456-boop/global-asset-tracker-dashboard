# 公开部署说明

目标：把本地 Streamlit 仪表盘部署成可通过公网 URL 访问的网站。

## 当前推荐

优先推荐 Render Docker 部署。原因：

- 项目已经包含 `Dockerfile` 和 `render.yaml`。
- 云端会长期运行，不依赖本机开机。
- 后续可以升级到持久磁盘或数据库。

Streamlit Community Cloud 也可用，适合快速上线，但需要通过 GitHub 仓库部署。

## 数据边界

不要上传这些目录：

- `data/raw/`
- `data/inbox/`
- `.streamlit/secrets.toml`

可以上传：

- `data/processed/signals.sqlite`

说明：`signals.sqlite` 是处理后的仪表盘数据。公开网站访问者能看到网页展示出的指标和排名。若不希望任何人直接看，应设置访问密码。

## 可选访问密码

本项目支持可选密码。

本地测试可创建：

```text
.streamlit/secrets.toml
```

内容：

```toml
ASSET_TRACKER_PASSWORD = "你的密码"
```

云端部署时不要上传 `secrets.toml`，应在平台环境变量或 Secrets 中设置：

```text
ASSET_TRACKER_PASSWORD=你的密码
```

如果不设置该变量，网站默认公开访问。

## 方案 A：Streamlit Community Cloud

适合：最快拿到公开 URL。

步骤：

1. 创建 GitHub 仓库。
2. 把项目根目录 `工程内容` 下的代码推送到仓库。
3. 确认仓库包含：
   - `streamlit_app.py`
   - `requirements.txt`
   - `.streamlit/config.toml`
   - `src/`
   - `pages/`
   - `data/processed/signals.sqlite`
4. 打开 Streamlit Community Cloud。
5. 选择仓库、分支和入口文件：

```text
streamlit_app.py
```

6. 如需密码，在 Streamlit 的 Secrets 中添加：

```toml
ASSET_TRACKER_PASSWORD = "你的密码"
```

7. 部署完成后，把生成的网址发给别人。

## 方案 B：Render Docker

适合：更接近长期网站。

步骤：

1. 创建 GitHub 仓库并推送项目。
2. 打开 Render，创建 Web Service。
3. 连接 GitHub 仓库。
4. 选择 Docker 部署；Render 会读取根目录 `Dockerfile`。
5. 如需密码，在 Environment Variables 添加：

```text
ASSET_TRACKER_PASSWORD=你的密码
```

6. 部署完成后使用 Render 分配的 `onrender.com` 网址。

项目也包含 `render.yaml`，可以用 Render Blueprint 方式创建服务。

## 每日更新云端数据

当前 SQLite 是随代码部署的静态数据文件。每日更新流程：

1. 本地把当天 Excel/PDF 放入 `data/inbox/YYYY-MM-DD/`。
2. 运行导入，更新 `data/processed/signals.sqlite`。
3. 提交并推送新的 `data/processed/signals.sqlite`。
4. 云平台自动重新部署或手动触发部署。

后续如果需要“只发文件，不重新部署代码”，应升级为远程数据库或云端上传后台。
